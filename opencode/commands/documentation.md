# Documentation Review and Update

Review and update the project documentation to ensure it's comprehensive, accurate, and user-friendly.

## Instructions for Agent

When this command is run, follow this workflow:

1. **Identify documentation files**:
   - Look for README.md, CONTRIBUTING.md, API docs, user guides, etc.
   - Check for inline code documentation (docstrings, comments)

2. **Review for accuracy**:
   - Verify examples match current code implementation
   - Check that API signatures are up-to-date
   - Ensure configuration examples are correct
   - Validate command-line examples work as shown

3. **Review for clarity**:
   - Check for confusing or ambiguous language
   - Ensure proper structure and flow
   - Verify code examples are well-commented
   - Check for broken links or references

4. **Review for completeness**:
   - Identify missing documentation for new features
   - Check if installation/setup steps are complete
   - Verify all public APIs are documented
   - Ensure troubleshooting section exists

5. **Suggest improvements**:
   - Recommend better examples where applicable
   - Suggest additional sections if needed
   - Propose clearer wording for complex concepts
   - Identify opportunities for diagrams or visuals

6. **Update as needed**:
   - Fix outdated information
   - Add missing documentation
   - Improve clarity and structure
   - Ensure consistency across all docs

## Documentation Standards

- Use clear, concise language
- Include practical examples
- Keep information up-to-date with code
- Follow markdown best practices
- Include table of contents for long docs
- Add code snippets with syntax highlighting
- Provide troubleshooting guidance where relevant
